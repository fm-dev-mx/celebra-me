# API Security + Response Contract

## Standard Server Response Shape

All API routes return one of two shapes (defined in `src/lib/rsvp/core/http.ts`):

**Success** (`ApiSuccess<T>`):

```typescript
{ success: true, data: T, meta?: { page, perPage, total } }
```

**Error** (`ApiErrorResponse`):

```typescript
{ success: false, error: { code: string, message: string, details?: unknown } }
```

Generated by `jsonResponse(payload, status)` for success, `errorResponse(error)` for errors.

## Client API Result Shape

The client-side fetch layer (`src/lib/api-client-shared.ts`) uses a different shape:

```typescript
// Success
{ data: T, ok: true, status: number }
// Error
{ ok: false, status: number, code: string, message: string, details?: unknown }
```

`ApiResult<T>` is the union. Used by `DashboardApiClient` in `src/lib/dashboard/api-client.ts`.

## ApiError / errorResponse Behavior

`ApiError` (`src/lib/rsvp/core/errors.ts`) carries `status`, `code`, `message`, and optional
`details`.

`errorResponse()` in `src/lib/rsvp/core/http.ts`:

- **ApiError input**: logs only status >= 500, returns the structured error preserving
  code/message/details
- **Empty object input** (e.g. `{}`): treated as bad request, returns 400
- **Fallback** (Error, string, or unknown): logs, returns 500 with `internal_error` code

Convenience wrappers: `unauthorizedResponse()`, `badRequest()`, `forbidden()`, `conflict()`,
`internalError()`.

Error codes are typed as `ApiErrorCode` — see the source file for the full union.

## Expected Admin Route Composition

Admin API routes follow this call order:

```
requireAdminStrongSession(request) → requireAdminRateLimit(request, operation) → handler logic → jsonResponse/errorResponse
```

### requireAdminStrongSession

Defined in `src/lib/rsvp/auth/authorization.ts`. Guards, in order:

1. SuperAdmin role check (403 `forbidden` if not admin)
2. MFA evidence on access token AMR claim
3. Dev MFA bypass (only when `isDevMfaBypassEnabled` + superadmin)
4. Fresh MFA required check (`REQUIRE_FRESH_MFA_FOR_ADMIN` env var)
5. Trusted device cookie verification (`sb-trust-device` cookie, HMAC-verified)
6. Fail with 403 if none pass

### requireAdminRateLimit

Defined in `src/lib/rsvp/security/admin-rate-limit.ts`. Operation keys and limits are in the
`RATE_LIMITS` map in the same file. Entity ID is `operation:${userId || hashedIp}`. Backend is
either Upstash (distributed) or in-memory (local), resolved via `rate-limit-provider.ts`.

## CSRF Lifecycle

Full implementation in `src/lib/rsvp/security/csrf.ts`.

| Step     | What happens                                       | Detail                                                                                                         |
| -------- | -------------------------------------------------- | -------------------------------------------------------------------------------------------------------------- |
| Generate | `randomBytes(32)` → base64url                      | Stateless, no DB storage                                                                                       |
| Store    | SHA-256 hash in `csrf-token` cookie                | httpOnly, secure (prod), SameSite=Strict, path=/, 24h expiry                                                   |
| Expose   | `<meta name="csrf-token">` in HTML                 | Read by client via `src/lib/csrf.ts`                                                                           |
| Validate | Header `X-CSRF-Token` → hash → timing-safe compare | Only for POST/PUT/PATCH/DELETE. Missing cookie = skip (no session). Missing header = fail (prod) or warn (dev) |
| Clear    | `cookies.delete(CSRF_COOKIE_NAME)`                 | On logout                                                                                                      |

## What Must Not Be Bypassed

- **CSRF**: All state-changing requests (POST, PUT, PATCH, DELETE) to authenticated routes must
  include `X-CSRF-Token`. Exceptions: `/api/webhook`, `/api/stripe`, `/api/supabase` (see
  `shouldSkipCsrfValidation`).
- **Admin strong session**: Every admin API route must call `requireAdminStrongSession` before the
  handler.
- **Rate limiting**: Every admin API route must call `requireAdminRateLimit` with the appropriate
  operation key.

## Out of Scope

This rule covers API response contracts, route composition, CSRF, strong-session requirements, and
rate-limit expectations.

Low-level session/cookie infrastructure is intentionally out of scope. Before changing access-token
resolution, refresh-token behavior, MFA cookies, trusted device cookies, or session cookie builders,
create or update a dedicated auth/session rule anchored to `src/lib/rsvp/auth/`.
